# TODO Template

Choose one of two styles based on task complexity:

---

## Style 1: Checklist (Simple Tasks)

Use this for straightforward tasks with clear steps.

```markdown
# [Task Name]

**Related Global Task:** [Link or description from TODO.md]
**Status:** In Progress / Completed

## Goal
Brief description of what needs to be done and why.

## Steps
- [ ] Step 1
- [ ] Step 2
- [ ] Step 3
- ✅ Completed step example

## Notes
- Important considerations
- Decisions made

## Completion Checklist
- [ ] All tests passing
- [ ] Code linted and formatted
- [ ] Documentation updated
- [ ] Changes committed
```

---

## Tips

1. **Choose the right style:**
   - Checklist → Quick features, bug fixes, simple tasks
   - Detailed → Refactoring, architecture, complex changes

2. **Mark completed items:**
   - Use ✅ emoji at the start of completed items
   - In checklists: `- ✅ Done` or `- [x] Done`
   - In detailed: `✅ **Completed**` at section start

3. **Update as you go:**
   - Mark items complete immediately after finishing
   - Add notes about issues or decisions
   - Keep status current

4. **Delete when done:**
   - Remove file after task completion
   - Or archive if you want to keep history
